# Document writing guidelines

Use this file to guide how the AI writes documents in Document mode. Edit it to match your style and requirements.

## Structure

- Start with a clear title (single `#` heading).
- Use headings to create a logical hierarchy: `##` for main sections, `###` for subsections.
- Keep paragraphs short (2–4 sentences) for readability.
- Use lists where order or bullets help (steps, options, features).

## Style

- Write in clear, direct language.
- Prefer active voice unless passive is more natural.
- Use consistent terminology; define acronyms on first use.

## Formatting

- Use **bold** for strong emphasis and *italic* for terms or light emphasis.
- Use code backticks for commands, names, and technical terms.
- Add links where they add value: `[text](url)`.

## Output

- Reply only with the document body in Markdown. No preamble, no "Here is your document", no commentary outside the content.
